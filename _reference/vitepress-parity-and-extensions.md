This project targets high visual and interaction parity with [VitePress](https://vitepress.dev/), while keeping configuration and authoring idiomatic to [Jekyll](https://jekyllrb.com/).

## VitePress parity targets

The theme intentionally mirrors:

- top nav, mobile nav screen, and social links
- left sidebar + right outline behavior
- home layout structure (hero + feature cards)
- doc footer behavior (edit link, previous/next pager, last-updated label)
- appearance switcher (`auto -> dark -> light`)
- search trigger UX (`/`, `Ctrl/Cmd+K`, `Cmd+K`)
- code block UX (copy buttons, language labels, file title bars/icons)

## Jekyll-first differences (intentional)

These are deliberate departures from VitePress internals:

- configuration lives in `_config.yml` under `jekyll_vitepress`
- nav/sidebar/social/version data live in `_data/*.yml`
- syntax highlighting uses Rouge themes (`jekyll_vitepress.syntax.*`)
- last-updated timestamps are filled via Jekyll plugin hooks
- extension points are include hooks instead of Vue slots:
  - `_includes/jekyll_vitepress/head_end.html`
  - `_includes/jekyll_vitepress/doc_footer_end.html`
  - `_includes/jekyll_vitepress/layout_end.html`

## Extras added on top

This theme includes optional extras that are not baseline VitePress core behavior:

- `jekyll_vitepress.github_star` navbar GitHub star widget with live star count
- simple local-search index from `search.json` generated by Jekyll content
- data-driven version selector from `_data/versions.yml` (`current: auto` supported)

## Practical guidance

If you want strict VitePress-like output, keep optional extras disabled and rely on default structure/styling.
If you want productized docs behavior, enable extras case by case in `_config.yml`.
